❯ what does this project do

⏺ This is a Flask-based Expense Tracker web app, structured as a step-by-step
  student project. Here's what it currently has and what's planned:

  Working now:
  - Landing page (/)
  - Register page (/register)
  - Login page (/login)

  Placeholder routes (not yet implemented):
  - /logout — user sign-out (Step 3)
  - /profile — user profile (Step 4)
  - /expenses/add — add an expense (Step 7)
  - /expenses/<id>/edit — edit an expense (Step 8)
  - /expenses/<id>/delete — delete an expense (Step 9)

  It also has a database/ module (SQLite likely), static CSS/JS, and HTML
  templates with a base layout. The app is meant to be built out incrementally —
   authentication, user profiles, and full CRUD for expenses.

❯ what techstack does this project use                                          
  Read 2 files (ctrl+o to expand)

⏺ Backend:
  - Python + Flask 3.1 — web framework
  - SQLite — database (via Python's built-in sqlite3)
  - Werkzeug — password hashing / request utilities

  Frontend:
  - HTML (Jinja2 templates)
  - CSS (static/css/style.css)
  - Vanilla JavaScript (static/js/main.js)

  Testing:
  - pytest + pytest-flask

  No ORM (raw SQL), no frontend framework — intentionally minimal for a learning
   project.

  │   ├── login.html          # Login form
  │   └── register.html       # Register form
  │
  ├── static/
  │   ├── css/style.css       # App-wide styles
  │   └── js/main.js          # Client-side JavaScript
  │
  ├── requirements.txt        # Python dependencies
  ├── .gitignore
  └── venv/                   # Virtual environment (not committed)

  How it fits together:
  - app.py imports from database/ and renders templates from templates/
  - Templates extend base.html for shared layout
  - static/ files are served directly to the browser
  - database/db.py will hold all SQLite logic once implemented
